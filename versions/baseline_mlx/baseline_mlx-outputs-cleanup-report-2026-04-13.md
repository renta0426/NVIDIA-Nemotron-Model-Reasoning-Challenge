## 上位注記

本報告書は、評価・提出契約の唯一の基準である **`README.md`** に基づいて作成した。最終提出物は **`submission.zip`** であり、README に記載された `max_lora_rank=32`, `max_tokens=7680`, `top_p=1.0`, `temperature=0.0`, `max_num_seqs=64`, `gpu_memory_utilization=0.85`, `max_model_len=8192` を前提に扱う。

今回の目的は、`baseline_mlx/outputs` 配下の全トップレベル実験を棚卸しし、**ベースライン 1 本 + 現状ベスト 2 本**のみを残して、それ以外を削除できる状態に整理することだった。実験内容は本レポートへ集約し、`outputs` 側には最小限の実体だけを残す。

## スコアとフロンティア

### 現状ベストとして残す raw run

- run root: `nemotron_sft_lora_with_cot_v2_mlx_stagefreeze_v2_stage25_attention_qkvo_reanchor_o30best_proxybench30ao_b10_t10_g15_u15_lr1e6_len1024_from_proxymiss_v1`
- `readme_local320 = 235/320 = 0.7344`
- `leaderboard_proxy_v1_set = 131/200 = 0.6550`

### raw proxy の最高観測値

outputs 走査で最も高い raw `leaderboard_proxy_v1_set` は、`reprobridge28` 系で **`0.6600`** だった。ただし `readme_local320` は **`0.7281`** で、保持対象に選んだ best raw run (`0.7344`, `0.6550`) より local320 が低い。  
そのため keep set は **proxy 単独最大**ではなく、README local320 と proxy_v1 の両立で選んだ。

### baseline anchor

`nemotron_sft_lora_with_cot_v2_mlx_notebook_original_fullrun_v2` は、現行 outputs の score JSON だけでは値を直接再抽出できなかったが、リポジトリ履歴上の baseline anchor であり、比較の出発点として残す。今回の削除判断は **baseline を必ず 1 本残す**方針に従った。

| `other` | 7 | artifact root, repro helper root, local_best helper root など。Git-visible ledger が残るため削除可。 |
